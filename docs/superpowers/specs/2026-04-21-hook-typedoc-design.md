# Design: TypeDoc Auto-Generation for @guoba-ai/hook

## Goal

Add automatic TypeDoc documentation generation for the `@guoba-ai/hook` package, making it a top-level sibling to `@guoba-ai/utils` in the docs sidebar.

## Approach

Two separate TypeDoc configs (Approach A) — one per package, each with its own entry points, output directory, and independent TypeDoc run.

## TypeDoc Configuration

### typedoc-utils.json (renamed from typedoc.json)

- Entry points: `../../packages/guoba-utils/src/{array,guard,object,string,types}.ts`
- tsconfig: `../../packages/guoba-utils/tsconfig.json`
- Output: `content/docs/utils/`
- publicPath: `/docs/utils/`
- All other settings (plugins, formatting) unchanged

### typedoc-hooks.json (new)

- Entry points: `../../packages/guoba-hook/src/{useToggle,useDebounce,useThrottle,usePrevious,useMount,useUnmount}.ts`
- tsconfig: `../../packages/guoba-hook/tsconfig.json`
- Output: `content/docs/hooks/`
- publicPath: `/docs/hooks/`
- Same plugins and formatting settings as utils config

## Content Structure

```
content/docs/
├── index.mdx              (Getting Started — unchanged)
├── meta.json              (updated: top-level lists utils + hooks)
├── utils/                 (was api/, generated by typedoc-utils.json)
│   ├── index.mdx          (auto-generated module index)
│   ├── meta.json          (auto-generated by postprocess)
│   ├── array/
│   ├── guard/
│   ├── object/
│   ├── string/
│   └── types/
└── hooks/                 (new, generated by typedoc-hooks.json)
    ├── index.mdx          (auto-generated module index)
    ├── meta.json          (auto-generated by postprocess)
    ├── useToggle.mdx
    ├── useDebounce.mdx
    ├── useThrottle.mdx
    ├── usePrevious.mdx
    ├── useMount.mdx
    └── useUnmount.mdx
```

## Post-Processing

Update `typedoc-postprocess.mjs`:

- Change hardcoded `content/docs/api` to process both `content/docs/utils` and `content/docs/hooks`
- Same logic for each: flatten `functions/` and `type-aliases/` subdirectories, fix internal links, generate `meta.json` per module directory

## Build Scripts

```json
{
  "typedoc": "typedoc --options typedoc-utils.json && typedoc --options typedoc-hooks.json && node typedoc-postprocess.mjs",
  "dev": "typedoc --options typedoc-utils.json && typedoc --options typedoc-hooks.json && node typedoc-postprocess.mjs && next dev",
  "build": "typedoc --options typedoc-utils.json && typedoc --options typedoc-hooks.json && node typedoc-postprocess.mjs && next build"
}
```
